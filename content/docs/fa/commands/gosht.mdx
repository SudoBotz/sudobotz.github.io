## تنظیم کانال‌های شنود

### افزودن کانال شنود

<Callout type="info" title="تنظیم کانال لاکر">
برای تنظیم کانالی که ربات پیام‌های لاکر را شنود کند:
```
/stp gosht add channel:#locker1
```

**توضیح:**
- هر زمان کاربری در کانال مشخص شده ایتم گوشت یا ماهی اضافه کند، ربات آن را حساب می‌کند
- محدودیتی در تعداد کانال‌های شنود وجود ندارد
- می‌توانید چندین کانال مختلف اضافه کنید

**مثال:**
```
/stp gosht add channel:#locker1
/stp gosht add channel:#locker2
/stp gosht add channel:#warehouse
```
</Callout>

### حذف یک کانال شنود

<Callout type="info" title="حذف کانال خاص">
برای حذف یک کانال خاص از لیست شنود:
```
/stp gosht remove channel:#locker1
```

این دستور فقط کانال مشخص شده را از لیست حذف می‌کند و سایر کانال‌ها همچنان فعال می‌مانند.
</Callout>

### حذف تمام کانال‌ها

<Callout type="warn" title="پاک کردن همه کانال‌های شنود">
برای پاک کردن تمام کانال‌های شنود:
```
/stp gosht clear
```

⚠️ **هشدار:** این دستور تمام کانال‌های شنود را حذف می‌کند! ربات دیگر هیچ کانالی را شنود نخواهد کرد.
</Callout>

## تنظیم کانال خروجی

### تنظیم کانال لاگ

<Callout type="info" title="کانال ارسال لاگ">
برای تنظیم کانالی که لاگ‌های خرید در آن ارسال شود:
```
/stp gosht set_output channel:#restorant
```

**توضیح:**
- هر زمان کاربری ماهی یا گوشت در لاکر بگذارد، پیام لاگ در این کانال ارسال می‌شود
- این قابلیت برای نظارت لحظه‌ای بر خریدها مفید است
- تنظیم این کانال اختیاری است

**نمونه لاگ:**
```
DEP: Amir_Kenzo | Mahi_Sardine:1x | ⁠╭🔴︴𝗟𝗼𝗰𝗸𝗲𝗿𝟭⁠ ✅
```
</Callout>

### حذف کانال خروجی

<Callout type="warn" title="پاک کردن کانال لاگ">
برای پاک کردن کانال خروجی لاگ:
```
/stp gosht clear_output
```

بعد از اجرای این دستور، دیگر لاگ‌ها در هیچ کانالی ارسال نمی‌شوند (اما شمارش ایتم‌ها همچنان ادامه دارد).
</Callout>

## مشاهده گزارش‌ها

### گزارش کلی کاربران

<Callout type="info" title="لیست کاربران و مبالغ">
برای مشاهده لیست کلی کاربرانی که ایتم گذاشته‌اند:
```
/rs
```

**نمونه خروجی:**
```

Farmer_Ali: 4x | $8,500
```

**توضیحات:**
- `6x` = تعداد کل ایتم‌ها
- `$26,000` = مبلغ کل قابل پرداخت

## ریست کردن ایتم‌ها

<Callout type="warn" title="پاک کردن ایتم‌های یک کاربر">
برای ریست کردن ایتم‌های یک کاربر خاص بعد از پرداخت:
```
/rs-reset ic_name:Amir_Kenzo confirms:y
```

**⚠️ نکات بسیار مهم:**
1. **تایید اجباری**: برای تایید عملیات حذف، **حتماً** باید `confirms:y` ارسال شود
2. **غیرقابل بازگشت**: این عملیات غیرقابل بازگشت است! قبل از اجرا مطمئن شوید
3. **پرداخت اول**: حتماً قبل از ریست کردن، مبلغ را به کاربر پرداخت کنید
4. **بررسی دقیق**: با دستور `/rs-items` جزئیات را قبل از ریست بررسی کنید

**مثال فرآیند کامل:**
```bash
# 1. مشاهده لیست کل
/rs

# 2. بررسی جزئیات یک کاربر
/rs-items ic_name:Amir_Kenzo

# 3. پرداخت $26,000 به کاربر (خارج از ربات)

# 4. ریست کردن ایتم‌ها
/rs-reset ic_name:Amir_Kenzo confirms:y
```


</Callout>

## نحوه استفاده (گام به گام)

<Callout type="info" title="راهنمای سریع راه‌اندازی">
**مرحله 1: تنظیم اولیه**
```bash
# افزودن کانال‌های لاکر
/stp gosht add channel:#locker1
/stp gosht add channel:#locker2

# تنظیم کانال لاگ (اختیاری)
/stp gosht set_output channel:#restorant

# بررسی تنظیمات
/stp gosht list
```

**مرحله 2: نظارت خودکار**
- ربات به صورت خودکار ایتم‌های گذاشته شده را شمارش می‌کند
- در صورت تنظیم کانال خروجی، لاگ‌ها ارسال می‌شوند

**مرحله 3: مشاهده گزارش‌ها**
```bash
# لیست کلی
/rs

# جزئیات یک کاربر
/rs-items ic_name:Amir_Kenzo
```

**مرحله 4: پرداخت و ریست**
```bash
# بعد از پرداخت مبلغ به کاربر
/rs-reset ic_name:Amir_Kenzo confirms:y
```
</Callout>

## خلاصه دستورات

<Callout type="info" title="جدول دستورات">
| دستور | کاربرد | نوع |
|-------|---------|------|
| `/stp gosht add channel:#xxx` | افزودن کانال شنود | تنظیمات |
| `/stp gosht remove channel:#xxx` | حذف کانال شنود خاص | تنظیمات |
| `/stp gosht clear` | حذف تمام کانال‌های شنود | تنظیمات |
| `/stp gosht set_output channel:#xxx` | تنظیم کانال خروجی | تنظیمات |
| `/stp gosht clear_output` | حذف کانال خروجی | تنظیمات |
| `/stp gosht list` | نمایش تنظیمات | تنظیمات |
| `/rs` | گزارش کلی کاربران | گزارش |
| `/rs-items ic_name:xxx` | جزئیات ایتم‌های کاربر | گزارش |
| `/rs-reset ic_name:xxx confirms:y` | ریست ایتم‌های کاربر | مدیریت |
</Callout>

## نکات مهم و توصیه‌ها

<Callout type="warn" title="نکات کلیدی">
**امنیت و دقت:**
1. ✅ همیشه قبل از ریست، مبلغ را پرداخت کنید
2. ✅ با دستور `/rs-items` جزئیات را بررسی کنید
3. ⚠️ ریست کردن غیرقابل بازگشت است

**بهینه‌سازی:**
- 📊 برای نظارت بهتر، کانال خروجی تنظیم کنید
- 📝 یک لیست از پرداخت‌ها در خارج از ربات نگه دارید

**محدودیت‌ها:**
- ❌ محدودیتی در تعداد کانال‌های شنود نیست
- ❌ محدودیتی در تعداد کاربران نیست
- ✅ فقط ایتم‌های مجاز لیست شده حساب می‌شوند
</Callout>
